# Environment Setup Guide

## 🔐 Admin Credentials Setup

Your admin credentials are now stored securely using environment variables instead of being hardcoded in the source code.

### Step 1: Generate Credentials

Run the setup script to generate admin credentials:

```bash
npx tsx scripts/setup-admin.ts
```

This will output environment variables to add to your `.env.local` file.

### Step 2: Create Environment File

Create a `.env.local` file in your project root with the following variables:

```env
# Admin Authentication
ADMIN_EMAIL=admin@gdgoc.com
ADMIN_PASSWORD=admin904
JWT_SECRET=your-super-secret-jwt-key-change-this-in-production

# Database Configuration (if using)
DATABASE_URL=your-database-connection-string

# Google Sheets API (if using)
GOOGLE_SHEETS_PRIVATE_KEY=your-google-sheets-private-key
GOOGLE_SHEETS_CLIENT_EMAIL=your-google-sheets-client-email
GOOGLE_SHEETS_SPREADSHEET_ID=your-google-sheets-spreadsheet-id

# Email Configuration (if using)
SMTP_HOST=your-smtp-host
SMTP_PORT=587
SMTP_USER=your-smtp-username
SMTP_PASS=your-smtp-password
```

### Step 3: Install Dependencies

Make sure you have the required dependencies:

```bash
npm install jsonwebtoken @types/jsonwebtoken
```

### Step 4: Restart Your Development Server

After adding the environment variables, restart your development server:

```bash
npm run dev
```

## 🔒 Security Features Implemented

- ✅ JWT token authentication
- ✅ HTTP-only secure cookies
- ✅ Input sanitization and validation
- ✅ Rate limiting (5 attempts per minute)
- ✅ XSS protection
- ✅ SQL injection protection
- ✅ CSRF protection via secure cookies
- ✅ Admin route protection middleware

## ⚠️ Important Security Notes

1. **Never commit `.env.local` to version control**
2. Change the admin password in the setup script before running
3. Use a strong, unique JWT_SECRET in production
4. Enable HTTPS in production
5. Consider using Redis for rate limiting in production
6. Regularly rotate admin credentials
7. Monitor login attempts and failed authentications
8. Use environment variables for all sensitive data
9. Implement proper logging for security events

## 🚀 Production Deployment

For production deployment:

1. Set up environment variables in your hosting platform
2. Use a strong, unique JWT_SECRET
3. Enable HTTPS
4. Set up proper logging
5. Configure rate limiting (Redis recommended)
6. Set up monitoring for failed login attempts
7. Regular security audits
8. Keep dependencies updated
